# Clase 2 - Investigación de APIs y Selección de Stack
## Sistema Multi-Red Social con LLM

**Fecha:** Martes 12 de noviembre de 2025

---

## Propósito de la sesión

Esta clase está dedicada principalmente a las **exposiciones de los estudiantes**. Cada estudiante presenta su investigación sobre las APIs de redes sociales, su comparativa de LLMs y la propuesta de arquitectura técnica para el proyecto.


## PRESENTACIÓN  

Al final de las exposiciones, el docente presenta brevemente la tarea para la próxima clase.

**Tarea para el 14 nov - Prototipo de Adaptación con LLM:**

Construir un sistema que tome contenido original y lo adapte automáticamente para las 5 redes sociales usando el LLM seleccionado.

**Entregable:**
- Sistema funcionando localmente
- Adaptación para Facebook, Instagram, LinkedIn, TikTok y WhatsApp
- Documentación de prompts utilizados
- Demo en vivo con 3 casos de prueba
- Exposición de 15 minutos sobre el desarrollo

Ver detalles completos en [TAREAS-Y-ENTREGABLES.md](../../TAREAS-Y-ENTREGABLES.md) - Clase 2.

## Preguntas Frecuentes

**P: ¿Qué pasa si no pude crear una app de Meta?**
R: Documenta el proceso que seguiste, los problemas encontrados y muestra screenshots del intento. No penaliza si está bien documentado.

**P: ¿Debo tener código funcionando para esta clase?**
R: No. Esta clase es solo investigación y documentación. El código comienza en la clase del 14 nov.

**P: ¿Puedo cambiar el stack después?**
R: Sí, pero deberás justificar el cambio. Es mejor decidir bien ahora.

**P: ¿Cuánto detalle debe tener mi diagrama de arquitectura?**
R: Debe mostrar los componentes principales (Frontend, Backend, LLM, APIs, Base de datos, Colas) y cómo se conectan.

**P: ¿Debo incluir costos en mi análisis?**
R: Sí, especialmente para LLMs y APIs que cobran (WhatsApp/Twilio).
